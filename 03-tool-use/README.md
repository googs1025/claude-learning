# 第三章：工具使用 (Tool Use / Function Calling)

## 核心概念

工具使用（Tool Use）是 Claude 最强大的能力之一。它允许 Claude 调用你定义的外部工具来获取信息或执行操作，
从而突破纯文本对话的限制，与真实世界进行交互。

## 工作流程

```
用户提问
   │
   ▼
┌─────────────────────┐
│  1. 定义工具        │  ← 你告诉 Claude 有哪些工具可用
│     (Tool Definition)│     每个工具有名称、描述、参数定义
└─────────┬───────────┘
          │
          ▼
┌─────────────────────┐
│  2. Claude 分析     │  ← Claude 阅读用户的问题
│     决定是否调用工具 │     判断是否需要使用工具来回答
└─────────┬───────────┘
          │
          ▼
    ┌─────┴─────┐
    │ 需要工具？ │
    └─────┬─────┘
     是 ↙     ↘ 否
       ▼        ▼
┌──────────┐  ┌──────────────┐
│ 返回     │  │ 直接回复文本  │
│ tool_use │  │ (end_turn)   │
│ 请求     │  └──────────────┘
└────┬─────┘
     │
     ▼
┌─────────────────────┐
│  3. 你执行工具      │  ← 你的代码执行实际操作
│     (Your Code)     │     如查询数据库、调用API等
└─────────┬───────────┘
          │
          ▼
┌─────────────────────┐
│  4. 返回工具结果    │  ← 把执行结果发回给 Claude
│     (Tool Result)   │     使用 tool_result 消息
└─────────┬───────────┘
          │
          ▼
┌─────────────────────┐
│  5. Claude 整合回复 │  ← Claude 根据工具结果
│     (Final Answer)  │     生成最终的自然语言回答
└─────────────────────┘
```

## Agentic Loop（代理循环）

在复杂场景中，Claude 可能需要多次调用工具才能完成任务。这就需要一个循环：

```
发送消息 → 检查 stop_reason
                │
         ┌──────┴──────┐
         │             │
    "tool_use"     "end_turn"
         │             │
    执行工具        完成！
    返回结果        输出回复
    继续循环
```

## 工具选择模式 (Tool Choice)

| 模式 | 说明 | 使用场景 |
|------|------|----------|
| `auto` | Claude 自行决定是否调用工具 | 默认模式，最常用 |
| `any` | Claude 必须调用至少一个工具 | 强制使用工具 |
| `tool` | Claude 必须调用指定的工具 | 指定调用某个工具 |

## 关键理解

1. **工具只是描述**：你只是告诉 Claude 工具的名称、用途和参数格式，实际执行由你的代码完成
2. **Claude 不执行代码**：Claude 只是决定调用哪个工具、传什么参数，真正的执行在你这边
3. **JSON Schema**：工具参数使用 JSON Schema 定义，Claude 会严格按照 schema 生成参数
4. **循环是关键**：实际应用中，一次对话可能需要多轮工具调用，循环处理是核心模式
